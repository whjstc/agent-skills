# Product Brand Naming Scorecard

Use this scorecard when ranking finalists. Score 1-5 unless a field is marked as a risk label.

| Dimension | What to Check |
| --- | --- |
| Category clarity | Does the name hint at the SaaS category, buyer problem, or technical job? |
| Brand distinctiveness | Is it more ownable than a plain descriptor? |
| Memorability | Can a buyer remember it after hearing it once? |
| Pronunciation | Can it be said clearly in sales calls and podcasts? |
| Spelling | Can people type it after hearing it? |
| Domain strength | Is `.com` available or acquirable? Are fallback TLDs acceptable? |
| Search uniqueness | Are exact-match search results sparse and non-confusing? |
| Trademark risk | Low / medium / high based on obvious same-category conflicts. |
| Roadmap fit | Will the name still work after adjacent features launch? |
| Claim-safety | Does the name avoid overpromising legal, financial, security, or compliance outcomes? |

## Recommendation Rules

- Prefer a slightly less clever name with a clean `.com` and low conflict risk.
- Avoid names that only work if the user understands a pun.
- Avoid names that require unusual capitalization to be readable.
- For regulated or compliance-adjacent SaaS, avoid names that imply certification, proof, guarantee, legality, or official approval unless that is actually the product.
- Treat "no obvious conflict found" as a screening result, not a legal conclusion.
